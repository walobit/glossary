DataCamp Glossary

absolute path
A path that always points to the same location. Absolute paths always start with the root directory.
blob
A "binary large object", i.e., a file of any kind in any format.
branch
A "parallel universe" in a version control repository. Changes made in one branch are isolated from changes made in other branches.
clone
A local copy of a version control repository.
comma-separated values (CSV)
A textual format for tabular data in which columns are separated by commas and records by newlines. There are many variations on CSV, differing primarily in how they handle quoted strings and empty columns.
command-line flag
An optional argument to a shell command that modifies its behavior. Most single-letter command-line flags in Unix begin with a single dash; longer flags usually begin with double dashes. Either type may take arguments, as in -n 3.
command-line shell
An interactive text-based program that enables a user to interact with the operating system. A shell typically only has a handful of built-in commands, and relies on external programs to perform most operations.
comment
Text that is included in a command or program, but ignored by the computer. Comments in the Unix shell begin with the # character; comments in languages like C and Java may use delimiters like /* and */.
commit
Either the act of making a permanent record of the state of a version control repository, or such a record itself.
conflict
Overlapping differences between the changes made concurrently by two different users or in two different branches of a version control repository.
current working directory
The default directory associated with a running program; in the shell, the directory the user is "in".
diff
The difference between one state of a version control repository and another, e.g., the changes that have to be made to get from one state to the next.
environment variable
A global named value associated with the shell that is inherited by all processes. Environment variables are typically used to store things like the user's home directory. See also shell variable.
filesystem
A way of organizing the files and metadata stored by a computer. The Unix filesystem is (mostly) hierarchical: directories are nested to form a tree, and files live in directories.
filesystem URLs
A URL of the form file:///some/absolute/path, sometimes used to refer to a version control repository on the computer where the command is being run.
global settings
Settings or preferences that apply to all of a user's projects.
hash
A short bit string generated from the contents of files in a deterministic but unpredictable way. Files that are exactly the same have the same hash; Two different files are almost certain to have different hashes.
hash function
A function that takes any data as input and generates a hash of that data.
home directory
The root of a user's personal section of a filesystem.
local settings
Settings or preferences that apply to a single project.
log
The historical list of all commit messages provided when making changes to a version control repository.
log message
The comment associated with a specific change in a version control repository.
loop
A control flow construct that instructs a computer to repeat a set of operations zero or more times.
merge
To combine and reconcile the changes made in two different branches of a version control repository.
nested repository
A repository located within another repository.
page
To display a screenful of a file at a time.
parent directory
The directory that contains the one in question. Every directory has a parent directory except the root directory of the filesystem.
path
A textual representation of a location in a filesystem. Paths can be absolute or relative.
pipe
A communication channel in the Unix shell that connects the output of one program to the input of another. Pipes are written with vertical bars, as in a | b.
prompt
A message printed by the shell to signal that it is awaiting input. The prompt can be as simple as a single $ character, or include status information like the time and current working directory.
pull
To download and merge changes from a remote version control repository.
push
To send changes to a remote version control repository.
redirection
Sending the standard output of a program to a file, or reading the standard input of a program from a file. The former is written as a > file, and the latter as a < file.
relative path
A path that must be interpreted from a starting point. For example, ../a/b.txt means "the file b.txt in the directory a in the parent directory of the current directory".
remote
A bookmark in a version control repository that identifies some other repository.
repository
A directory that stores both user-generated content and the metadata used by a version control system.
repository status
The current state of a version control repository (e.g., whether any files have uncommitted changes).
root directory
The folder in the filesystem that contains everything else either directly or indirectly.
shell script
A set of shell commands saved in a file for re-execution; equivalently, a program that is executed by the shell.
shell variable
A local name-value variable associated with a single process. See also environment variable.
staging area
Part of the metadata of a Git version control repository that records what changes will be saved in the next commit.
system settings
The settings and preferences that apply to all users and their projects on a particular computer.
tab completion
Having the shell fill in the rest of a command or path when the tab key is pressed.
tree
In the Git version control system, a description of the files and directories present at a particular moment.
undo
To reset a version control repository to a previous state.
version control system
A software tool for recording and managing changes to a set of files.
wildcard
A single placeholder character that can match zero or more actual characters in text. For example, the wildcard ? in the Unix shell matches any single actual character.
